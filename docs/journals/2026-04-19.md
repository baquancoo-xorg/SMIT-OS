# Journal - 2026-04-19

## Responsive Design System Plan Created

**Context:** User reported UI/UX issues across devices - elements too large, inconsistent button sizes, varying page heights causing poor experience on iPad Pro M1, iPhone 13-16, and Desktop (1366px-4K).

**Decision:** Selected Design Tokens + Viewport Units approach over Container Queries (too much refactor) and Adaptive Layouts (too much maintenance).

**Key Technical Choices:**
- `dvh`/`dvw` + `clamp()` for responsive sizing
- Viewport-fit layout (no page-level scroll)
- Internal scroll within components
- 9 breakpoints targeting specific devices

**Plan Structure:**
| Phase | Focus | Days |
|-------|-------|------|
| 1 | Foundation (tokens, AppLayout) | 1-2 |
| 2 | TechBoard pilot | 3-4 |
| 3 | Rollout 12 pages | 5-6 |
| 4 | Polish + docs | 7+ |

**Risks Identified:**
- Safari dvh quirks → test early on real devices
- DnD may break with layout changes → minimal DOM changes
- Many Kanban cards → may need virtualization
