## Notes & Observations

1. `list_daily_reports` formats output as Markdown table, not JSON — initial E2E script tried `JSON.parse` on it and skipped `get_report_by_id` auto-capture. Reran with explicit UUID from DB. **Not a tool bug** — Markdown is the intended user-facing format.
2. `revenue_summary` reuses `/api/dashboard/overview` endpoint (revenue scope folded into dashboard) — matches plan-level design decision.
3. MCP api-client retry policy: 5xx triggers exponential backoff (500ms, 1500ms); 4xx fast-fails. With CF 502 ~900ms each, worst-case end-to-end ~5s — well within 12s SLA.
4. axios timeout 10s. If CF tunnel itself is down (not just app), worst case = 10s × 3 attempts = 30s. Edge case; acceptable.
