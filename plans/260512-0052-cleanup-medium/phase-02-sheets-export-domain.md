## Implementation Steps

1. **DB backup (MANDATORY):**
   ```
   mkdir -p backups
   docker exec smit_os_db pg_dump -U postgres smitos_db > backups/pre-sheets-drop-$(date +%Y%m%d-%H%M).sql
   ```
   Verify file size > 0.
2. **Re-verify `server.ts` line numbers** with grep (lines may drift):
   ```
   grep -n "sheets-export\|google-oauth\|googleOAuthService\|sheetsExportService" server.ts
   ```
3. **Edit `server.ts`** — remove the 4 imports (31, 32, 34, 39) and 4 mount/init lines (126, 127, 131, 152, 153). Be sure no dangling references remain (`googleOAuthService` variable in particular).
4. **Edit `src/pages/Settings.tsx`** — apply 7 edits per researcher-02 Q2: remove tab entry, remove conditional render branch, remove lazy import, remove related state hooks, remove any prop wiring.
5. **Edit `src/components/settings/index.ts`** — remove `sheets-export-tab` re-export.
6. **Delete backend files** (8 paths in delete list). Use `git rm` for staged removal.
7. **Delete UI file:** `git rm src/components/settings/sheets-export-tab.tsx`.
8. **Grep orphan refs:**
   ```
   grep -rn "GoogleIntegration\|SheetsExportRun\|sheets-export\|googleOAuthService" --include="*.ts" --include="*.tsx" --exclude-dir=node_modules
   ```
   Result must be empty (except schema lines about to be removed).
9. **Edit `prisma/schema.prisma`** — delete `model GoogleIntegration` and `model SheetsExportRun` blocks. Confirm no other model references them via relation.
10. **Run migration:**
    ```
    npx prisma migrate dev --name drop-sheets-export-models
    ```
    Inspect generated SQL — should be 2 `DROP TABLE`.
11. **Regenerate client:** automatic via `migrate dev`. Verify `node_modules/.prisma/client/index.d.ts` no longer mentions dropped models.
12. **Server smoke (interim):** `npm run dev` boots without sheets imports/routes.
13. **Commit:** `feat(cleanup): remove google sheets export domain + drop prisma models`.
