## Suggestions (non-blocking)

### S1 ‚Äî DailySync: `submission` sort accessor returns raw string
**File:** `src/pages/DailySync.tsx:135-137`
```ts
case 'submission':
  return row.createdAt;
```
The `submission` column displays `<SubmissionBadge createdAt={r.createdAt} />` which is conceptually a timestamp. Returning raw string sorts lexically ‚Äî which, for ISO-8601 timestamps from API, *is* chronological, so behaviorally fine. But for consistency with the `createdAt` case three lines below (which wraps in `new Date()`), consider unifying. Cosmetic only.

**Suggested:**
```ts
case 'submission':
  return new Date(row.createdAt);
```

### S2 ‚Äî WeeklyCheckin: `week` accessor returns raw string
**File:** `src/pages/WeeklyCheckin.tsx:91-92`
```ts
case 'week':
  return row.weekEnding;
```
`weekEnding` is `string` per `types/index.ts:71`. ISO-format strings sort correctly lexically. If API ever changes to ISO-8601 with timezone offset (e.g. `2026-05-04T00:00:00.000Z`), still fine. Lower urgency than S1; behavior is correct today.
